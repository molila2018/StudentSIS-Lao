# StudentSIS Lao — Windows Installer

This folder builds a one-file Setup `.exe` that installs the StudentSIS Lao
application onto a Windows PC. The installer is created with
[Inno Setup](https://jrsoftware.org/isinfo.php), which is free, mature, and
standard for Windows desktop apps.

## What the installer does

| | |
|---|---|
| **Bundled runtime** | Self-contained .NET 8 — no separate runtime install needed. |
| **Install location** | `%LOCALAPPDATA%\Programs\StudentSIS_Lao` (per-user, no UAC) by default. The user can opt to install for all users at the elevation prompt, which switches to `C:\Program Files\StudentSIS_Lao`. |
| **Start Menu** | Adds two entries: `StudentSIS Lao` and `StudentSIS Lao (ພາສາລາວ)`, plus an uninstall shortcut. |
| **Desktop shortcut** | Optional — a tick-box on the wizard. |
| **Templates** | The school's Excel/Word templates ship inside the installer (`Templates\` folder). |
| **Database** | `sis_lao.db` is created on first launch next to the EXE. The installer **never** touches it, so upgrades and re-installs preserve all student data. |
| **Upgrade** | Run a newer Setup `.exe` on top of an existing install — files overwrite, database survives. |
| **Uninstall** | Standard Windows uninstaller. Leaves `sis_lao.db` behind (safer default — the school can wipe it by hand if they really want to). |

## One-time prerequisites on the build machine

1. **.NET 8 SDK** — <https://dotnet.microsoft.com/download/dotnet/8.0>
2. **Inno Setup 6.x** — <https://jrsoftware.org/isinfo.php> (accept the default install path)

That's it. Nothing else is required.

## Building the installer

From a `cmd` prompt at the project root, run:

```cmd
installer\build.cmd
```

Or just double-click `installer\build.cmd` in Windows Explorer.

The script does three things:

1. `dotnet publish` — Release, self-contained, win-x64 → outputs to `publish\`
2. Locates `ISCC.exe` (Inno Setup compiler)
3. Compiles `StudentSIS_Lao.iss` → outputs to `installer-output\`

When it finishes you'll have:

```
installer-output\StudentSIS_Lao_Setup_v1.0.0.exe   (~80–150 MB)
```

Hand this single file to the school — they double-click it to install.

## Bumping the version

Edit two places when releasing:

1. `StudentSIS.csproj` → `<Version>1.0.0</Version>` (and `<FileVersion>` / `<AssemblyVersion>`)
2. `installer\StudentSIS_Lao.iss` → `#define MyAppVersion "1.0.0"`

Then re-run `build.cmd`. The installer file name embeds the new version automatically (`StudentSIS_Lao_Setup_v1.0.1.exe`).

## What the user sees

1. Double-click `StudentSIS_Lao_Setup_v1.0.0.exe`.
2. Standard Inno wizard: Welcome → install location → tasks → install → finish.
3. Final wizard page offers a "Launch StudentSIS Lao" check-box.
4. App appears under Start Menu → `StudentSIS Lao`.
5. To uninstall: Settings → Apps → `StudentSIS Lao 1.0.0` → Uninstall.

## Optional polish (later)

These aren't required for a working installer but make it more professional:

- **App icon**: add an `.ico` file at project root, then set `<ApplicationIcon>` in
  `StudentSIS.csproj` and `SetupIconFile=..\<icon>.ico` in the `[Setup]` block.
- **Code signing**: a real Authenticode certificate suppresses SmartScreen
  warnings. Add `SignTool=...` to `[Setup]` and configure a signtool entry.
- **Bundle Phetsarath OT font**: drop the .ttf into `installer\fonts\` and add a
  `[Files]` line that installs it into `{fonts}` with `onlyifdoesntexist`.
  The app already falls back to Leelawadee UI when Phetsarath isn't installed,
  so this is just a quality-of-output improvement.
- **License page**: add `LicenseFile=LICENSE.txt` to `[Setup]` to show terms
  during install.

## Regenerating the user manual

A Lao-language user manual (PDF) is generated by a separate script:

```cmd
installer\generate-manual.cmd
```

This builds the integration-test project (which contains the manual generator),
writes a temporary `.docx` using OpenXML, and converts to PDF via Microsoft
Word's COM automation (or LibreOffice headless as a fallback). Output:

```
installer\User-Manual.pdf
```

To edit the manual content, modify `tests\IntegrationTests\UserManual.cs` —
all chapters, headings, and steps live in the `BuildDocx()` method. Rebuild
with the script above. Word is used as the renderer rather than iTextSharp
because iTextSharp 5 can't position Lao combining vowel marks correctly.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `dotnet publish failed` | .NET 8 SDK missing | Install from <https://dotnet.microsoft.com/download> |
| `Inno Setup compiler ISCC.exe not found` | Inno Setup not installed or not in `PATH` | Install Inno Setup 6 — the script also looks in the default install dirs automatically |
| `Compression error` during install | Antivirus quarantined the temp extraction | Whitelist the install location or run the installer as admin |
| App launches but Lao text shows as boxes | Phetsarath OT font missing on the target PC | Install Phetsarath OT system-wide, OR rely on the built-in fallback to Leelawadee UI |
| App can't write the database | Installed to Program Files without admin write rights | Re-install per-user (the default) — the database lives next to the EXE and needs a writable folder |
